Extra Credit Process Work

Usability Testing Report

Interviewees: Max, Ella (friends)

Both interviewees fit the target audience because they are unfamiliar with Blackjack.

Summary of Key Findings

  • Game navigation (buttons) were straightforward
  • Overall UI was clean and easy to understand, but more context for rules is needed for users unfamiliar with Blackjack

Task:

Users: Manny, Max, Ella

  • Start game
  • Place bet (choose number of carrots to bet)
  • Complete one full game of blackjack. Either “hit” (deal another card) or “stand” (don’t deal another card), and interact with end game modal.
  • Optional: continue playing more rounds

Bugs:

  • Numbers were difficult to read with pixelated font
  • Couldn’t continue or start new game after losing all carrots
  • Unsure what bets were for
  • Unsure about how to play game (rules, number when dealer stops)
